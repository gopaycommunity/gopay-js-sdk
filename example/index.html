Run yarn example from the repo root — it builds the SDK and starts the dev server automatically. Credentials and base URL are read from sdk/.env.e2e (GP_GW_JS_SDK_CLIENT_ID, GP_GW_JS_SDK_CLIENT_SECRET, GP_GW_JS_SDK_BASE_URL) and fall back to process.env in CI. Opening via file:// still works but env values won't be injected. In production use https://gopaycdn.com/js-sdk/gopay-sdk.min.js.

auth.authenticate()

POST /oauth2/token — obtain an access token using client credentials.

—

auth.setRefreshToken() — browser flow

Simulates the browser client pattern: a server-side step obtains a token pair using the credentials above, then a fresh SDK instance is seeded with only the refresh token (no credentials in the browser). The first API call (payments.create()) automatically exchanges the refresh token for an access token. 

—

Uses the Client ID / Secret / GoID fields above. Authenticate first is not required — this flow is self-contained.

payments.create()

POST /eshops/{goid}/payments — create a payment session. Authenticate first to obtain a token.

—

On success, the response includes gw_url — redirect the customer there to complete payment.

payments.charge()

POST /payments/{id}/charge — charge using a card token.

—

Use the Payment ID from payments.create() output. Card token obtained via cards.createToken().
