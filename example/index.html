auth.authenticate() (server-only — never call from a browser)

POST /oauth2/token — obtain an access token using client credentials. Keep client_id and client_secret on the server.

—

auth.issueClientToken() — server side

Obtain a fresh token pair for a browser client without affecting the server session. Authenticate first, then issue a token with a narrower scope if needed. The result is auto-filled into the browser flow below.

—

Requires auth.authenticate() to have been called first.

auth.setClientToken() — browser flow

Simulates the browser client pattern: a fresh SDK instance receives both tokens from the server (no credentials in the browser). The client_id is extracted automatically from the JWT.

—

Client credentials must never leave the server. Run auth.issueClientToken() above to populate the fields, or paste tokens obtained from your own server endpoint.

payments.create()

POST /eshops/{goid}/payments — create a payment session. Authenticate first to obtain a token.

—

On success, use the returned id to charge the payment via card token, Apple Pay, Google Pay, etc.

payments.getStatus()

GET /payments/{id} — retrieve the current state of a payment (CREATED, PAID, CANCELED, etc.).

—

Google Pay

Step 1: payments.getGooglePayInfo() — fetch config, render Google Pay button.
Step 2: Click the Google Pay button — user selects a card. Token is prefilled into payments.charge() below. 

—

Requires a payment created with Google Pay as an allowed instrument. In sandbox the Google Pay sheet runs in TEST mode — no real card data is used.

Apple Pay

Step 1: payments.getApplePayInfo() — fetch config, render Apple Pay button.
Step 2: Click the Apple Pay button — merchant validation runs automatically via payments.startApplePaySession(). Token is prefilled into payments.charge() below. 

—

Requires a payment created with Apple Pay as an allowed instrument. ApplePaySession: native on Safari; simulated by a dev polyfill in other browsers. PaymentRequest: shown on Chrome/Edge when the browser supports Apple Pay — triggers a cross-device QR code flow authenticated via iPhone or Apple Watch.

payments.getQRPaymentInfo()

GET /payments/{id}/qr-payment/info — fetch QR code (base64) and recipient info.

—

QR codes are returned base64-encoded. The available field depends on currency: spayd (CZK), paybysquare/sepa (EUR), mnb_qr (HUF).

Card Pay

Step 1: payments.create() fills in the Payment ID below.
Step 2: Click Open Card Payment — cards.mountCardForm() fetches the hosted iframe URL from the API, mounts it, and tokenizes. Token is prefilled into payments.charge() below.

payments.charge()

POST /payments/{id}/charge — submit the payment instrument token obtained from Google Pay, Apple Pay, or Card Pay above.

No instrument prefilled — complete a payment flow above, or enter a card token manually.
—

payments.getChargeState()

GET /payments/{id}/charge — retrieve the current state of a charge (REQUESTED, PROCESSING, SUCCEEDED, FAILED, etc.).

—

cards.getDetails()

GET /cards/tokens/{card_id} — retrieve details of a stored permanent card token (masked PAN, expiry, scheme, fingerprint). Requires card:read scope.

—

cards.deleteCard()

DELETE /cards/tokens/{card_id} — permanently remove a stored card token. Returns no content on success.

—

recurrences.createRecurrence()

POST /eshops/{goid}/recurrences — create a recurring payment agreement. The returned id is auto-filled into the status / start / next / stop fields below.

—

recurrences.recurrenceStatus()

GET /recurrences/{rec_id} — retrieve the current state of a recurrence (NEW, STARTED, STOPPED, etc.).

—

recurrences.startRecurrence()

POST /recurrences/{rec_id}/start — trigger the first charge of a recurrence that is in the NEW state.

—

recurrences.recurrenceNext()

POST /recurrences/{rec_id}/next — charge the next instalment of a STARTED recurrence. Amount and order number are optional overrides.

—

recurrences.stopRecurrence()

DELETE /recurrences/{rec_id} — permanently stop a recurrence. This cannot be undone.

—

links.createPaymentLink()

POST /eshops/{goid}/links — create a shareable payment link. The returned id and URL are auto-filled below.
