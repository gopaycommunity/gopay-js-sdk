payments.create()

POST /eshops/{goid}/payments — create a payment session. Returns id and payment_secret; both are auto-filled into the sections below.

—

payments.getStatus()

GET /payments/{id} — retrieve the current state of a payment (CREATED, PAID, CANCELED, etc.).

—

payments.getGooglePayInfo()

GET /payments/{id}/google-pay/info — fetch the PaymentsClient config and paymentDataRequest needed to render a Google Pay button.

—

Requires a payment created with Google Pay as an allowed instrument. The full button flow (render + charge) is in the Browser SDK section below.

Apple Pay

Step 1: payments.getApplePayInfo() — fetch config, render Apple Pay button.
Step 2: Click the Apple Pay button — merchant validation runs automatically via payments.startApplePaySession(). The charge fires inside onpaymentauthorized. 

—

Requires a payment created with Apple Pay as an allowed instrument. ApplePaySession: native on Safari; simulated by a dev polyfill in other browsers.

payments.getQRPaymentInfo()

GET /payments/{id}/qr-payment/info — fetch QR code (base64) and recipient info.

—

QR codes are returned base64-encoded. The available field depends on currency: spayd (CZK), paybysquare/sepa (EUR), mnb_qr (HUF).

payments.charge()

POST /payments/{id}/charge — submit a payment instrument token obtained from Google Pay, Apple Pay, or a saved card token. Google Pay and Apple Pay flows above prefill the instrument automatically. For one-time card payments use the Browser SDK's Card Pay section below.

No instrument prefilled — complete a Google Pay or Apple Pay flow above, or enter a card token manually.
—

payments.getChargeState()

GET /payments/{id}/charge — retrieve the current state of a charge (REQUESTED, PROCESSING, SUCCEEDED, FAILED, etc.).

—

cards.getDetails()

GET /cards/tokens/{card_id} — retrieve details of a stored permanent card token (masked PAN, expiry, scheme, fingerprint). Requires card:read scope.

—

cards.deleteCard()

DELETE /cards/tokens/{card_id} — permanently remove a stored card token. Returns no content on success.
