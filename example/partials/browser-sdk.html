Browser SDK — Step 2: sdk.attachPayment({ paymentId, paymentSecret })

Your server creates a payment and returns id and payment_secret to the browser. Call sdk.attachPayment() to bind the browser SDK to that payment. Run sdk.createPayment() above to auto-fill these fields.

sdk.mountCardForm(container, options)

Mounts the GoPay-hosted card encryption iframe. Two flows: return-payload — resolves with { encryptedPayload }, auto-filled into the Browser charge section below; forward the payload to your server and call sdk.tokenizeEncryptedCard(). direct-charge — the SDK charges the attached payment in the browser and returns the terminal PaymentChargeStatusResponse; requires attachPayment() to have been called first. 3DS is handled automatically per the chosen mode.

Test card: 4111 1111 1111 1111 — any future expiry (e.g. 12/30), any 3-digit CVV.

sdk.mountGooglePayButton(container, options?)

mountGooglePayButton() — auto-fetches config, renders the official Google Pay button, handles the payment sheet, charges, and polls to completion. Requires an attached payment. 

—

Requires a payment created with Google Pay as an allowed instrument. In sandbox the Google Pay sheet runs in TEST mode — no real card data is used.

sdk.mountApplePayButton(container, options?)

mountApplePayButton() — auto-fetches config, renders the <apple-pay-button>, handles merchant validation and the payment sheet, charges, and polls to completion. Requires an attached payment. 

—

Requires a payment created with Apple Pay as an allowed instrument. ApplePaySession: native on Safari; simulated by a dev polyfill in other browsers.

sdk.getQRPaymentInfo(format?)

Attach the payment and call getQRPaymentInfo() — returns QR code (base64) and recipient info.

—

QR codes are returned base64-encoded. The available field depends on currency: spayd (CZK), paybysquare/sepa (EUR), mnb_qr (HUF).

sdk.chargePayment(params)

Charge the attached payment using the encrypted card payload from the card form above. sdk.attachPayment() is called automatically after sdk.createPayment(). Polls sdk.getChargeState() automatically until SUCCEEDED or FAILED.

—

sdk.getChargeState()

GET /payments/{id}/charge — retrieve the current charge state (REQUESTED, PROCESSING, SUCCEEDED, FAILED, ACTION_REQUIRED) for the attached payment.

—

sdk.getStatus()

GET /payments/{id} — retrieve the full payment details for the currently attached payment. Useful to inspect state, amounts, and instrument details at any point after attachPayment().
